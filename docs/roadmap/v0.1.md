# v0.1 — Base Infrastructure And Documentation

Status: completed.

## Goal

Prepare the repository foundation for `max_secretary`: documentation structure, production deployment basics, backend skeleton, Docker Compose runtime and CI/CD foundation.

## Scope

- Root project documentation: `README.md`, `CHANGELOG.md`, `VERSION`.
- Documentation tree under `docs/`.
- VPS preparation documentation.
- SSH deploy user and key-based access documentation.
- Production Docker Compose file.
- Backend Dockerfile and `.dockerignore`.
- Nginx reverse proxy config.
- `.env.example` and `.gitignore` for secrets hygiene.
- GitHub Actions backend CI.
- Manual GitHub Actions deploy workflow.
- Initial backend FastAPI app with `/api/health`.
- Initial Alembic setup and PostgreSQL-only configuration.

## Infrastructure Decisions

- Project services run through Docker Compose.
- PostgreSQL and Redis are not installed directly on the VPS.
- Nginx is containerized.
- Secrets are loaded from `.env`.
- PostgreSQL and Redis are not published externally.

## Outcomes

- Repository has a deployable base.
- `/api/health` is available through nginx.
- Production runtime can be started with:

```bash
docker compose -f docker-compose.prod.yml up -d --build
```
