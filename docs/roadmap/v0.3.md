## Goal

Add MAX Bot webhook command support without coupling task business logic to a concrete external webhook payload.

## Scope

- `POST /api/bot/max/webhook`.
- Normalized bot event schema.
- MAX-like event normalization.
- Command parser.
- Bot service connected to existing task services.
- Logging `MaxSender` stub.
- Bot command documentation.
- Bot webhook smoke test.

## Design Notes

- Parser does not access the database.
- Service layer resolves users/chats and executes commands.
- MVP commands use internal UUIDs for current chat/user when command execution needs existing backend entities.
- Assignee and observer matching for `/–∑–∞–¥–∞—á–∞` uses `display_name`.

## Constraints

- Real MAX API sending was not enabled in this release track.
- Inline buttons and native task cards were not implemented.
- Full MAX WebApp auth was not implemented.

## Outcome

The project gained a working bot-command adapter over the backend task workflow.
