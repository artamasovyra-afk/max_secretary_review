# v1.0.1 — Pilot Hardening

Status: planned.

## Goal

Package post-`v1.0.0` hardening fixes and documentation cleanup into a patch release without changing the product scope.

## Planned Focus

- Documentation cleanup after audit.
- Deployment documentation deduplication.
- README and release docs alignment.
- Architecture docs completion.
- API routes snapshot refresh.
- Test inventory refresh.
- Worker lifecycle and healthcheck hardening.
- Production/offline compose healthcheck alignment.
- Nginx routing documentation alignment.
- Release gate tooling improvements.
- CI improvements for WebApp build, shell syntax and compose config.
- Backup/restore verification documentation.

## Candidate Fixes

- Mark legacy deployment docs as redirects or archive pages.
- Ensure offline compose and offline scripts use current release tags.
- Keep `docker-compose.offline.yml` aligned with production healthchecks.
- Avoid stale statements about MAX/Bitrix24 implementation status.
- Document production ref policy: immutable tag vs current `main`.

## Out Of Scope

- New business features.
- Full MAX WebApp auth.
- Automatic Bitrix24 sync.
- Two-way Bitrix24 sync.
- Real file upload storage.

## Exit Criteria

- Backend tests pass.
- WebApp build passes.
- Compose config passes for production and offline compose.
- Release smoke passes on VPS.
- No secrets, `.env`, backup files, Docker image tar files, wheels or npm cache are committed.
- `VERSION` and tag are updated only when the patch release is intentionally cut.
