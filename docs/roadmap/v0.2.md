# v0.2 — Backend Task MVP

Status: completed.

## Goal

Implement the backend task core for `max_secretary`.

## Scope

- Organizations.
- Users.
- Chats.
- Chat members.
- Tasks.
- Multiple assignees through `TaskAssignee`.
- Observers through `TaskObserver`.
- Comments.
- File metadata.
- Assignee responses.
- Requester acceptance and rejection.
- Inbox summary across chats.
- SQLAlchemy models and Alembic migrations.

## API Surface

Main API groups:

- `/api/organizations`
- `/api/users`
- `/api/chats`
- `/api/tasks`
- `/api/tasks/inbox/summary`

## Task Workflow

The backend supports:

- task creation;
- assignee and observer lists;
- status transitions;
- assignee response submission;
- requester accept/reject flow;
- comments and file metadata.

## Constraints

- No authorization was enforced broadly at this stage.
- MAX webhook, WebApp, Bitrix24 and AI were intentionally out of scope.

## Outcome

`max_secretary` gained the core domain model and task APIs needed for later bot, reminders and WebApp features.
