# v0.5 — WebApp MVP

Status: completed.

## Goal

Add a browser-based MVP interface for working with tasks.

## Scope

- React, TypeScript, Vite and Ant Design WebApp.
- Application layout with navigation.
- Dashboard page.
- Tasks list page.
- Task details page.
- Settings page shell.
- API client layer.
- Task creation modal.
- Comments UI.
- File metadata UI.
- Assignee response UI.
- Requester accept/reject UI.
- WebApp Dockerfile.
- WebApp service in production Docker Compose.
- Nginx routing for WebApp and backend API.
- WebApp smoke test.

## Main Pages

- Dashboard: unified inbox summary.
- Tasks: task list with filters.
- Task Details: full task card, comments, files, responses and actions.
- Settings: MVP shell for later configuration.

## Deployment

The `webapp` container serves static files through nginx. Public traffic still enters through the top-level `nginx` service.

## Constraints

- Full MAX WebApp authentication was not implemented.
- `user_id` was temporarily passed through query parameters and later moved toward AuthContext preparation.
- File upload stores metadata only.

## Outcome

The project became usable through a WebApp MVP, not only API and bot commands.
