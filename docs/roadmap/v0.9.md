# v0.9 — Closed Contour / Offline Bundle

Status: completed.

## Goal

Prepare `max_secretary` for closed-contour deployment where the production server must not download dependencies from the internet.

## Scope

- Offline delivery policy documentation.
- Python wheelhouse build script.
- Docker image save/load scripts.
- `docker-compose.offline.yml`.
- Release manifest generation.
- SHA256 checksum tooling.
- Offline bundle build script.
- Closed-contour install guide.
- Manual dependency update policy.

## Offline Bundle Structure

```text
vendor/
  python-wheels/
  npm-cache/
  docker-images/
  checksums/
  release/
```

`vendor/*` artifacts are not committed to git, except lightweight `.gitkeep` or README files when needed.

## Deployment Model

- Images are built and saved before delivery.
- Target contour loads Docker images from tar files.
- Offline compose uses `image:`, not `build:`.
- External integrations remain disabled unless explicitly configured in the target contour.

## Security

- Production servers do not pull packages from the internet.
- Release artifacts get manifest/checksum files.
- Dependency updates are manual and controlled.

## Constraints

- Building the offline bundle requires Docker on the build machine.
- Closed-contour acceptance still requires infrastructure-specific verification.

## Outcome

`max_secretary` gained a documented closed-contour delivery path and tooling for offline artifacts.
