# v0.4 — Reminders Scheduler

Status: completed.

## Goal

Implement the first reminders layer for task deadlines and daily summaries.

## Scope

- Reminder service.
- Reminder repository and schemas.
- Reminder rules API for tasks.
- Reminder rules API for chats.
- Overdue task detection.
- Waiting acceptance reminder flow.
- No-response-after-deadline reminder flow.
- Daily summary builder.
- Worker scheduler based on APScheduler.
- Logging `MaxSender` for reminder notifications.
- Reminder smoke test.
- Product documentation for reminders.

## Reminder Types

- `before_deadline`
- `at_deadline`
- `after_deadline`
- `no_response_after_deadline`
- `waiting_acceptance`
- `daily_summary`

## Runtime Behavior

- Scheduler runs in the `worker` container, not inside backend API workers.
- `REMINDERS_ENABLED=false` disables scheduler startup.
- Real MAX sending remains disabled unless explicitly configured.

## Constraints

- Celery/Celery Beat was not introduced.
- Reminder delivery is logged through the sender adapter unless real sending is enabled.

## Outcome

Tasks can be selected for reminder scenarios, overdue tasks can be marked, and the worker can run scheduled reminder jobs.
