# v0.8 — RBAC And WebApp Auth Preparation

Status: completed.

## Goal

Prepare the project for role-based access control and future MAX WebApp authentication.

## Scope

- RBAC policy documentation.
- Permission constants.
- `AuthContext`.
- `PolicyService`.
- Dev auth dependency through headers.
- RBAC protection for Bitrix24 sync endpoints.
- RBAC protection for Bitrix24 user mapping endpoints.
- WebApp AuthContext.
- Dev auth warning in WebApp.
- Documentation for future WebApp auth integration.

## Roles

- `member`
- `manager`
- `chat_admin`
- `super_admin`

## Protected Areas

The first protected endpoints were Bitrix24 sync and mapping endpoints because they can affect external systems or integration configuration.

## Dev Auth

MVP/dev auth supports headers such as:

- `X-User-Id`
- `X-Organization-Id`
- `X-Chat-Id`
- `X-Roles`

In production, `DEV_AUTH_ENABLED=false` is the safe default unless explicitly enabled for a controlled pilot.

## Constraints

- Full MAX WebApp authentication is not implemented.
- Dev auth headers are temporary.
- RBAC is not yet enforced across every task endpoint.

## Outcome

The project now has a clear policy layer and an auth context foundation for future production authentication.
