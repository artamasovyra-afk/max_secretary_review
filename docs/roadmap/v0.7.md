# v0.7 — Bitrix24 Connector MVP

Status: completed.

## Goal

Add one-way manual sync from `max_secretary` tasks to Bitrix24 without changing the core task workflow.

## Scope

- Bitrix24 integration settings.
- `IntegrationAccount` model.
- `BitrixTaskLink` model.
- `BitrixUserMapping` model and CRUD API.
- Bitrix24 REST client adapter.
- Bitrix24 task mapper.
- Manual sync service.
- Manual task sync endpoint.
- Task sync status endpoint.
- Retry failed sync endpoint.
- Bitrix24 connector smoke test in disabled mode.
- WebApp sync status card in Task Details.

## Public MVP API

- `POST /api/integrations/bitrix24/tasks/{task_id}/sync`
- `GET /api/integrations/bitrix24/tasks/{task_id}/status`
- `POST /api/integrations/bitrix24/retry-failed`
- `POST /api/integrations/bitrix24/user-mappings`
- `GET /api/integrations/bitrix24/user-mappings`

## WebApp

Task Details shows Bitrix24 sync status and allows manual sync. Task list does not show Bitrix24 status in v1.0.0 because a batch status endpoint is intentionally not part of the MVP.

## Security

- Bitrix24 webhook URL is treated as a secret.
- Real HTTP requests are disabled unless `BITRIX24_ENABLED=true` and webhook URL is configured.
- Sync errors are stored without exposing tokens or webhook URLs.

## Constraints

- Manual sync only.
- No automatic sync triggers.
- No two-way sync.
- No Bitrix24 import.
- No Bitrix24 task deletion.

## Outcome

The project gained a safe Bitrix24 Connector MVP with disabled-mode smoke coverage and visible task-level sync state.
