# RBAC Policy

Документ описывает текущую role model для backend API, MAX bot commands и WebApp.

## Roles

### `member`

Обычный пользователь чата.

- Создает задачи только самому себе.
- Видит свои задачи, а также задачи чата только при явной chat policy.
- Сдает отчеты по задачам, где он исполнитель.
- Не назначает задачи другим пользователям.
- Не создает group assignment.
- Не принимает и не отклоняет чужие отчеты.
- Не управляет участниками или настройками чата.

### `chat_admin`

Админ чата.

- Назначает задачи другим пользователям.
- Создает group assignment.
- Меняет исполнителей, наблюдателей и сроки задач в своем scope.
- Принимает и отклоняет отчеты.
- Пингует исполнителей.
- Управляет участниками, настройками чата и reminder rules.
- Управляет Bitrix24 mapping в рамках своей организации.

### `super_admin`

Супер-админ с глобальным доступом ко всем организациям, чатам, задачам и интеграциям.

## Removed Role

`manager` больше не является активной бизнес-ролью. Существующие `chat_members.role = 'manager'` мигрируются в `chat_admin`, потому что ранее `manager` использовался как управленческая роль задач.

Новые значения роли `manager` не принимаются в chat member schemas.

## Rules

1. Задачу видит `super_admin`, постановщик, исполнитель, наблюдатель или `chat_admin` в scope задачи.
2. `member` может создать задачу только с собой как единственным исполнителем.
3. Назначение другим, group assignment и управление участниками доступны только `chat_admin` и `super_admin`.
4. Отчет отправляет только исполнитель задачи.
5. Отчет принимает или отклоняет постановщик, `chat_admin` в scope задачи или `super_admin`.
6. Bitrix24 sync может запускать постановщик задачи, `chat_admin` в scope задачи или `super_admin`.
7. Bitrix24 mapping может менять `chat_admin` или `super_admin`.

## Auth Context

В `APP_ENV=local` и `APP_ENV=test` dev headers разрешены:

- `X-User-Id`
- `X-Organization-Id`
- `X-Chat-Id`
- `X-Roles`, например `member,chat_admin`

В `APP_ENV=production` dev headers отключены по умолчанию и не заменяют MAX WebApp session auth.
