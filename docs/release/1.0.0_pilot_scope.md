# 1.0.0 Pilot Scope

Документ фиксирует состав пилотной стабильной версии `max_secretary` 1.0.0.

## Цель Пилота

Пилотная версия должна подтвердить, что `max_secretary` закрывает основной сценарий контроля поручений из рабочих чатов: создание задач, назначение исполнителей, контроль ответа, приемка результата, свод задач и базовая интеграционная готовность.

## Входит В Пилот

### Backend

- задачи;
- несколько исполнителей;
- наблюдатели;
- комментарии;
- file metadata;
- ответы исполнителей;
- приемка и отклонение результата постановщиком;
- единый свод задач;
- reminders;
- MAX webhook adapter;
- Bitrix24 manual sync;
- RBAC/auth preparation.

### WebApp

- Dashboard;
- Tasks list;
- Task details;
- comments, files, responses and acceptance flows;
- Bitrix24 sync status in task details;
- dev auth context.

### Deployment

- Docker Compose production;
- VPS deployment;
- offline bundle docs and scripts.

## Не Входит В Пилот

- полноценная MAX WebApp auth;
- automatic Bitrix24 sync triggers;
- two-way Bitrix24 sync;
- Bitrix24 import;
- real file upload storage;
- production SSO;
- production-grade observability stack;
- full closed-contour certification.

## Pilot Stability Criteria

- Backend tests pass.
- WebApp build passes.
- Production Docker Compose starts successfully.
- Health endpoint returns `status=ok`.
- Core smoke tests pass on VPS.
- Secrets are not stored in git.
- External integrations can remain disabled without breaking task workflow.
