# Bot Commands

Документ описывает MVP-команды MAX bot adapter для проекта `max_secretary`.

## /задача

Формат:

```text
/задача <текст> | <исполнители> | <срок> | наблюдатели: <наблюдатели>
```

Назначение: создать задачу в текущем чате.

Примеры:

```text
/задача Подготовить отчет | Иван | 2026-05-20
/задача Подготовить отчет | Иван, Мария | 2026-05-20 | наблюдатели: Сергей, Анна
```

Ожидаемый результат: создается задача со статусом `new`, исполнителями из списка и наблюдателями, если они указаны.

Ошибки формата:

- не указан текст задачи;
- не указаны исполнители;
- срок не в формате `YYYY-MM-DD`;
- блок наблюдателей указан не как `наблюдатели: Сергей, Анна`.

## /задачи

Формат:

```text
/задачи
```

Назначение: показать активные задачи текущего чата.

Пример:

```text
/задачи
```

Ожидаемый результат: бот возвращает список задач чата, кроме задач в статусах `done` и `cancelled`.

Ошибки формата:

- команда не принимает аргументы.

## /мои_задачи

Формат:

```text
/мои_задачи
```

Назначение: показать задачи текущего пользователя.

Пример:

```text
/мои_задачи
```

Ожидаемый результат: бот возвращает задачи, где пользователь является исполнителем.

Ошибки формата:

- команда не принимает аргументы.

## /ответ

Формат:

```text
/ответ <task_id> <текст ответа>
```

Назначение: отправить ответ исполнителя по задаче.

Пример:

```text
/ответ 311ed506-64c4-4d4e-8851-5b92d6c21a8a Готов первый вариант отчета
```

Ожидаемый результат: создается `TaskResponse`, а исполнитель отмечается как ответивший.

Ошибки формата:

- не указан `task_id`;
- не указан текст ответа;
- `task_id` не является UUID.

## /готово

Формат:

```text
/готово <task_id> <текст ответа>
```

Назначение: отправить финальный ответ исполнителя по задаче.

Пример:

```text
/готово 311ed506-64c4-4d4e-8851-5b92d6c21a8a Отчет подготовлен
```

Ожидаемый результат: создается `TaskResponse`. Для MVP команда работает как `/ответ`.

Ошибки формата:

- не указан `task_id`;
- не указан текст ответа;
- `task_id` не является UUID.

## /принять

Формат:

```text
/принять <task_id> <response_id>
```

Назначение: принять результат по ответу исполнителя.

Пример:

```text
/принять 311ed506-64c4-4d4e-8851-5b92d6c21a8a 03491e85-765f-4be3-9ba8-41a14d41b6a9
```

Ожидаемый результат: `TaskResponse` получает статус `accepted`, задача переходит в `done`.

Ошибки формата:

- не указан `task_id`;
- не указан `response_id`;
- `task_id` или `response_id` не является UUID.

## /отклонить

Формат:

```text
/отклонить <task_id> <response_id> <комментарий>
```

Назначение: отклонить результат по ответу исполнителя с комментарием.

Пример:

```text
/отклонить 311ed506-64c4-4d4e-8851-5b92d6c21a8a 03491e85-765f-4be3-9ba8-41a14d41b6a9 Нужно добавить детализацию
```

Ожидаемый результат: `TaskResponse` получает статус `rejected`, задача возвращается в `in_progress`.

Ошибки формата:

- не указан `task_id`;
- не указан `response_id`;
- не указан комментарий;
- `task_id` или `response_id` не является UUID.

## Ограничения MVP

- Сопоставление пользователей по `display_name` временное.
- Реальный MAX Bot API sender будет подключен позже.
- Команды работают через webhook adapter.
