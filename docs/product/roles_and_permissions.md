# Roles And Permissions

Документ описывает актуальную модель ролей `Дьяк`.

## Роли

### `member`

Обычный пользователь чата.

- Создает задачи только самому себе.
- Смотрит свои задачи и доступные задачи чата, если политика чата это разрешает.
- Сдает отчеты по задачам, где он исполнитель.
- Не назначает задачи другим пользователям.
- Не создает групповые задачи.
- Не принимает и не отклоняет чужие отчеты.
- Не управляет участниками и настройками чата.

### `chat_admin`

Админ чата.

- Назначает задачи другим пользователям.
- Создает групповые задачи.
- Меняет и добавляет исполнителей.
- Переносит сроки и управляет задачами в своем chat/org scope.
- Принимает и отклоняет отчеты.
- Пингует исполнителей.
- Управляет участниками и настройками чата.
- Управляет Bitrix24 user mapping в рамках своей организации.

### `super_admin`

Супер-админ.

- Имеет полный доступ ко всем организациям, чатам, задачам и интеграциям.
- В policy определяется через `AuthContext.is_super_admin=true` или роль `super_admin`.

## Removed Role

`manager` удален из активной бизнес-логики. Миграция `20260525_020000_migrate_manager_role_to_chat_admin.py` переводит существующие строки `chat_members.role = 'manager'` в `chat_admin`, чтобы не потерять уже выданные управленческие права.

Новые API-запросы с ролью `manager` для участников чата отклоняются schema validation.

## Основные Правила

### Просмотр Задачи

Задачу может видеть:

- `super_admin`;
- постановщик задачи;
- исполнитель задачи;
- наблюдатель задачи;
- `chat_admin` в совпадающем chat/org scope;
- `member` в совпадающем scope, если настройки чата разрешают просмотр задач участниками.

### Создание Задачи

- `member` может создать задачу только с собой как единственным исполнителем.
- `chat_admin` и `super_admin` могут назначать задачи другим пользователям.
- Group assignment доступен только `chat_admin` и `super_admin`.

### Обновление Задачи

- Постановщик может обновлять свою задачу в пределах разрешенных backend-операций.
- Изменение исполнителей и наблюдателей требует `chat_admin` или `super_admin`.
- `chat_admin` управляет задачами в своем scope.
- `super_admin` управляет всем.

### Отчет Исполнителя

Отчет может отправить исполнитель задачи. Постановщик не отправляет отчет за исполнителя без отдельного бизнес-flow.

### Приемка И Отклонение

Принять или отклонить отчет может:

- постановщик задачи;
- `chat_admin` в scope задачи;
- `super_admin`.

### Bitrix24 Permissions

- Ручную синхронизацию по задаче может запускать постановщик, `chat_admin` в scope задачи или `super_admin`.
- Retry failed sync доступен `chat_admin` и `super_admin`.
- User mapping CRUD доступен `chat_admin` в рамках своей организации и `super_admin`.

## Dev Auth

В `APP_ENV=local` и `APP_ENV=test` dev auth headers разрешены.

Основные headers:

- `X-User-Id`
- `X-Organization-Id`
- `X-Chat-Id`
- `X-Roles`, например `member,chat_admin`

В `APP_ENV=production` dev auth headers отключены по умолчанию и не являются production-auth моделью.
