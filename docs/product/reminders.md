# Reminders

Документ описывает продуктовую логику напоминаний MVP для проекта `max_secretary`.

## Типы напоминаний

- `before_deadline` — напоминание до наступления срока задачи.
- `at_deadline` — уведомление в момент наступления срока.
- `after_deadline` — уведомление после истечения срока.
- `no_response_after_deadline` — уведомление, если после срока нет ответа исполнителя.
- `waiting_acceptance` — уведомление постановщику, если задача ждет приемки результата.
- `daily_summary` — ежедневная сводка задач пользователя.

## Сценарии

### Напомнить исполнителю до срока

Система находит активные задачи с ближайшим `deadline_at` и формирует напоминание для исполнителей. Задачи в статусах `done` и `cancelled` не участвуют.

### Уведомить в момент срока

Система формирует уведомление для задач, срок которых наступил в текущем scheduler window.

### Уведомить после срока

Система находит активные задачи с истекшим `deadline_at` и формирует уведомление о просрочке.

### Уведомить, если нет ответа исполнителя

Если срок истек, а обязательный исполнитель еще не отправил ответ, система формирует отдельное напоминание типа `no_response_after_deadline`.

### Уведомить постановщика, если задача ждет приемки

Когда задача находится в статусе `waiting_acceptance`, система формирует напоминание постановщику о необходимости принять или отклонить результат.

### Утренняя сводка задач

Система формирует `daily_summary` по задачам пользователя: назначенные задачи, созданные пользователем задачи, наблюдаемые задачи, задачи на сегодня, просроченные задачи и задачи, ожидающие реакции пользователя.

## Настройки

### На уровне задачи

Правило, привязанное к `task_id`, действует только для конкретной задачи. Такой уровень нужен для индивидуальных сроков, особых повторов и ручной настройки важных поручений.

### На уровне чата

Правило, привязанное к `chat_id`, используется как дефолт для задач чата. Такой уровень нужен, чтобы задать общую политику напоминаний для рабочего чата.

### Дефолтные настройки

Дефолтные настройки задаются на уровне приложения и worker scheduler:

- `REMINDERS_ENABLED` включает или выключает scheduler.
- `REMINDER_POLL_INTERVAL_SECONDS` задает интервал проверки due reminders.
- `DAILY_SUMMARY_TIME` задает время ежедневной сводки.

## Ограничения MVP

- Реальная отправка в MAX пока заменена `MaxSender`-заглушкой.
- `MaxSender` только логирует `chat_id`, `user_id`, текст сообщения и тип напоминания.
- Scheduler работает в отдельном worker-контейнере и не запускается в FastAPI backend API.
- Celery и Celery Beat могут быть подключены позже.
