# WebApp MVP

Документ описывает WebApp MVP для проекта `max_secretary`.

## Назначение WebApp

WebApp нужен для единого просмотра и контроля задач из разных рабочих чатов MAX. Интерфейс помогает быстро увидеть свои задачи, поставленные поручения, наблюдаемые задачи, просрочки, ответы исполнителей и задачи, ожидающие приемки постановщиком.

## Брендинг

В пилотном интерфейсе WebApp отображается под названием `Дьяк`. В боковом меню используется новый знак с дьяком, пером и конвертом, а для браузера добавлены favicon и app icons на основе нового брендинга.

Основные пункты меню переведены на русский язык:

- `Сводка`;
- `Задачи`;
- `Групповые поручения`;
- `Настройки`.

## UI polish for demo

Для промежуточной демонстрации WebApp приведен к стилю лаконичной служебной панели с MAX-акцентом:

- интерфейс использует логотип `Дьяк` и компактный знак с дьяком, пером и конвертом;
- верхний header выровнен так, чтобы название и подзаголовок не обрезались;
- основное меню отображается на русском языке;
- страница задач получила компактный header и KPI-панель;
- таблица задач стала плотнее и чище для регулярной рабочей проверки;
- страница задач адаптирована для узких экранов: фильтры переносятся, а таблица скроллится внутри своего блока;
- фильтры задач используют русские пользовательские подписи вместо технических `organization_id`, `chat_id` и похожих полей;
- статусы и приоритеты отображаются русскими подписями через компактные tags;
- тестовые smoke-задачи могут скрываться UI-фильтром для демонстрации без удаления данных;
- ошибка доступа к статусу Битрикс24 в карточке задачи показывается как спокойное информационное состояние, а не как технический raw error;
- Сводка без выбранного пользователя показывает понятный CTA и выбор пользователя для пилотного режима;
- Настройки оформлены как полезная пилотная заглушка с будущими разделами;
- состояния загрузки, пустого списка и ошибки сделаны дружелюбными для пользователя.

В рамках polish не добавлялись:

- канбан;
- графики;
- темная тема;
- новые UI-библиотеки.

Future note: если размер frontend bundle станет проблемой, стоит рассмотреть route-level code splitting для Ant Design и страниц WebApp. В рамках демонстрационного polish build-конфигурация не менялась.

## Страницы

### Сводка

Единый свод задач пользователя. Показывает задачи по основным категориям: мои задачи, поставленные мной, наблюдаемые, ожидающие моего ответа, ожидающие моей приемки, просроченные и задачи на сегодня.

### Задачи

Общий список задач с фильтрами по статусу, сроку, организации, чату, постановщику, исполнителю и наблюдателю. На странице доступно создание новой задачи.
Задачи показываются с коротким пользовательским номером `task_ref`, например `#1042`; внутренний UUID остается доступен в карточке для технической поддержки.

### Групповые поручения

Пилотный раздел для руководителя или администратора MAX-группы. Позволяет создать групповое поручение участникам конкретной группы, включить обязательный индивидуальный отчет и открыть отчет по выполнению.

Страница списка показывает:

- групповые поручения без загрузки отчетов по каждой строке;
- постановщика и его роль;
- группу;
- статус, срок и признак обязательного отчета;
- KPI: всего, выполнены, ожидают, просрочено.

Страница отчета показывает конкретного постановщика, роль, группу, срок, сводку выполнения и таблицу участников с индивидуальным статусом и текстом ответа. Если у группы несколько администраторов, в отчете видно, кто именно создал поручение.

### Карточка задачи

Карточка задачи с описанием, статусом, приоритетом, сроком, постановщиком, исполнителями, наблюдателями, комментариями, файлами metadata, ответами исполнителей и историей статусов.
В карточке задачи также отображается статус синхронизации с Битрикс24 и доступен ручной запуск синхронизации.
В заголовке и деталях карточки показывается короткий номер задачи, а UUID остается внутренним идентификатором существующих API routes.

### Настройки

Заготовка для будущих настроек WebApp, интеграций, профиля пользователя и параметров отображения.

## Основные сценарии

### Просмотр единого свода задач

Пользователь открывает страницу `Сводка` и видит задачи, собранные из разных чатов. WebApp восстанавливает сессию через `GET /api/auth/me`; если сессии нет, при открытии внутри MAX отправляет `initData` на `POST /api/auth/max-webapp/session`. Query parameter `user_id` не используется как production auth.

### Создание задачи

Пользователь открывает страницу `Задачи`, нажимает кнопку создания задачи, выбирает организацию, чат, постановщика, исполнителей, наблюдателей, срок, приоритет и правило завершения.

### Создание группового поручения

Пользователь с ролью `chat_admin` или `super_admin` открывает раздел `Групповые поручения`, выбирает организацию и чат, задает название, описание, срок и флаги обязательного отчета. После создания WebApp открывает отчет по поручению.

### Просмотр карточки задачи

Пользователь открывает задачу из `Сводки` или `Задач` и видит полную карточку с деталями, участниками и историей изменений статуса.

### Комментарий

Пользователь добавляет текстовый комментарий к задаче. Комментарий сохраняется как часть карточки задачи.

### Файл metadata

Пользователь добавляет metadata файла: имя, URL, storage key, MIME type и размер. Фактическая загрузка файла в MVP не выполняется.

### Ответ исполнителя

Исполнитель отправляет ответ по задаче. Backend переводит задачу в нужный статус согласно правилу завершения.

### Приемка постановщиком

Постановщик принимает ответ исполнителя. Задача переходит в статус `done`.

### Отклонение результата

Постановщик отклоняет ответ исполнителя с комментарием. Задача возвращается в рабочий статус для доработки.

### Синхронизация с Битрикс24

В карточке задачи пользователь видит статус синхронизации:

- `disabled` — интеграция выключена;
- `pending` — задача еще не синхронизирована;
- `synced` — задача связана с задачей Битрикс24;
- `error` — была ошибка синхронизации.

Доступные действия:

- `Синхронизировать с Битрикс24` — ручной запуск синхронизации;
- `Повторить синхронизацию` — повторный ручной запуск при ошибке.

Внешний URL задачи Битрикс24 может быть добавлен позже.

## Ограничения MVP

- MAX WebApp auth через `initData` и httpOnly session cookie подключен на backend и frontend.
- Прямое открытие в браузере без валидной session cookie или MAX `initData` показывает unauthorized state.
- `user_id` через query parameter допустим только для local/dev диагностики и не является production auth.
- Короткие номера задач уникальны внутри организации; глобальный UUID остается внутренним идентификатором.
- Файлы хранятся только как metadata.
- Реальная загрузка файлов будет реализована позже.
- Дизайн базовый и построен на Ant Design.
- Раздел `Групповые поручения` предназначен для пилотной демонстрации; UI шаблонов и scheduled tasks будет отдельной задачей.
- Список групповых поручений временно фильтрует `task_type=group_assignment` на frontend, пока нет отдельного backend list endpoint/filter.
- Bitrix24 sync status доступен в карточке задачи.
- Список задач не показывает Bitrix24 sync status в MVP.
- Автоматическая синхронизация с Битрикс24 не включена.
- Двусторонней синхронизации с Битрикс24 нет.
- Реальные Bitrix24 webhook URL не отображаются в WebApp.
